## Proje Konuları

Geliştireceğiniz projede yazılım kalite güvencesi ve testi kapsamında güncel bir odak alanı seçmeniz beklenmektedir (ör. web test otomasyonu, mobil test araçları, güvenlik açık tarayıcıları). Bazı konu örnekleri aşağıda verilmiştir (kendi konu önerinizi de verebilirsiniz):

1. Web Test Otomasyonu için Framework Geliştirme: Selenium / Playwright / Cypress vb. kullanarak, tekrar kullanılabilir bir test otomasyon framework’ü tasarlama, test raporlama, CI entegrasyonu.
1. Mobil Uygulama Test Otomasyonu: Android/iOS için Appium, Espresso vb. araçlarla test otomasyonu, farklı cihaz ve ekran çözünürlükleri için test senaryoları.
1. API / Servis Testi Otomasyonu: REST veya GraphQL servisleri için otomatik testler geliştirme.
1. Web Uygulamaları için Güvenlik Açığı Tarama Aracı İncelemesi: OWASP ZAP, Burp Suite vb. araçlardan birini seçerek bir web uygulaması üzerinde kullanma. Bulunan zafiyetlerin raporlanması ve sınıflandırılması (OWASP Top 10 ile ilişkilendirme).
1. DevOps Süreçlerinde Sürekli Entegrasyon ve Sürekli Test: GitLab CI / GitHub Actions / Jenkins kullanarak testlerin pipeline’a entegre edilmesi, her push’ta otomatik çalışan testler ve raporlama.
1. Container Tabanlı Uygulamalarda Test Yaklaşımları: Docker veya Kubernetes ortamında çalışan bir uygulamanın test edilmesi, test veri yönetimi, test ortamı izolasyonu
1. Kullanılabilirlik Testi ve Kullanıcı Deneyimi (UX) Değerlendirmesi: Belirli bir web veya mobil uygulama için görev tabanlı kullanılabilirlik testi yürütme, gözlem, anket ve ölçümlerle raporlama.
1. Yapay Zekâ Destekli Test Senaryosu Üretimi: Büyük dil modelleri (örn. ChatGPT, Copilot vb.) kullanarak gereksinim dokümanlarından otomatik test senaryoları üretme, üretilen test senaryolarının kapsam ve doğruluğunun değerlendirilmesi.
1. Makine Öğrenmesi ile Hata (Bug) Tahmini ve Sınıflandırma: Geçmiş hata kayıtları (issue tracker, bug report vb.) üzerinden makine öğrenmesi modeli eğiterek hangi modüllerde hata çıkma olasılığını tahmin etme. Modellin doğruluk, precision/recall vb. metriklerle değerlendirilmesi
1. Doğal Dil İşleme (NLP) ile Gereksinimlerden Otomatik Test Çıkarma: Metin hâlindeki gereksinim dokümanlarını NLP ile analiz ederek olası test durumlarını otomatik çıkarmaya yönelik prototip geliştirme, elde edilen testlerin insan uzmanlar tarafından hazırlanmış testlerle karşılaştırılması.
1. Büyük dil modellerinin (LLM’ler: GPT, Claude, Llama vb.) otomatik kod inceleme (code review) aracı olarak kullanılabilirliğini araştırma, belirli bir yazılım projesi veya kod tabanı üzerinde LLM’e kod inceleme yaptırma (ör. güvenlik açıkları, anti-pattern’ler, performans sorunları, kod kokuları).LLM’in bulgularını geleneksel statik analiz araçlarıyla (SonarQube, ESLint, PMD, Bandit vb.) karşılaştırma.

### 3.2 Kullanılan Teknolojiler ve Araçlar: Programlama dili, test kütüphanesi, çerçeve (framework), veri seti veya test ortamı konfigürasyonlarını açıklayın. Seçimlerini neden yaptığınızı teknik gerekçelerle destekleyin.

### 3.3 Uygulama Geliştirme: Geliştirdiğiniz sistemin adım adım nasıl çalıştığını açıklayın. Test senaryolarınızı, kurulum adımlarınızı, algoritmalarınızı veya otomasyon akışınızı sunun. Örnek kullanımlar, test çıktıları, örnek ekran görüntüleri / loglar eklenmelidir.
